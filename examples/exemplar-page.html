PubWeb Exemplar

Pages appear when the network agrees they exist.

You are viewing a page addressed by content hash, not by server path. PubWeb uses peers to store HTML payloads, a tracker to coordinate who has what, and a wrapper route that quietly resolves the page before rendering it.

Hash-addressed HTML Tracker-coordinated swarm Wrapper-based loading Peer-powered hosting

1. Publish

A peer receives HTML and computes SHA-256. That hash becomes the page address and content identity.

2. Announce

The peer announces inventory to the tracker, including hash and basic metadata such as the page title.

3. Resolve

When a user opens tracker/hash, the wrapper polls resolve until an active peer can serve the content.

4. Render

The wrapper loads page/hash into an isolated frame, so users see the site while the fetch process stays invisible.
